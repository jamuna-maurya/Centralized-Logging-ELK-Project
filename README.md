# Centralized Logging System using ELK Stack

## Project Overview

This project demonstrates a **Centralized Logging System** using the **ELK Stack (Elasticsearch, Logstash, Kibana)**. Logs generated by a Node.js application are collected, processed, indexed, and visualized through Kibana dashboards.

In modern production environments, applications run across multiple servers and containers. Centralized logging enables teams to aggregate logs in one place for monitoring, troubleshooting, and incident investigation.

## Installation

### Clone Repository

```bash
git clone https://github.com/<your-github-username>/Centralized-Logging-ELK-Project.git

cd Centralized-Logging-ELK-Project
```

### Start ELK Stack

```bash
docker compose up -d
```

### Verify Containers

```bash
docker ps
```

Expected containers:

```text
backend
elasticsearch
logstash
kibana
```

## Kibana Dashboards

The following dashboards were created:

### Error Count Over Time

Tracks application errors over time.

### Request Volume

Monitors total request count.

### Status Code Distribution

Visualizes HTTP response status codes.

### Endpoint Hits

Displays most frequently accessed endpoints.

### Log Severity Breakdown

Shows distribution of Info, Warning, and Error logs.
